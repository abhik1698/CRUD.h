Create Table Syntax

MySQL syntax:

CREATE TABLE tableName (column1 datatype, column2 datatype);

MySQL Shorthand syntax:

c tableName column1 column2

Drop Table Syntax

MySQL:

DROP TABLE tableName;

MySQL Shorthand:

dr tableName

Drop Schema Syntax

MySQL:

DROP DATABASE databaseName;

MySQL Shorthand:

r

Insert Row Syntax

MySQL:

INSERT INTO tableName VALUES( value1, value2 );

MySQL Shorthand:

i tableName value1 value2

Update Row Syntax

MySQL:

UPDATE tableName SET column = value where column1 = value1;

MySQL Shorthand:

u tableName column1 value1 column value

Delete Row Syntax

MySQL:

DELETE FROM tableName WHERE column = value;

MySQL Shorthand:

d tableName column1 value1

Truncate Table Syntax

MySQL:

TRUNCATE TABLE tableName;

MySQL Shorthand:

t tableName

MySQL Shorthand

SQL

  • SQL is a standard language for accessing and manipulating relational databases.

  • SQL stands for Structured Query Language.

  • SQL is used to update database by queries.

    • MySQL

  • MySQL is a Relational Database Management System.

  • MySQL is a database Software.

  • MySQL allows to handle & modify data in a database through tables.

    • MySQL Shorthand

  • MySQL Shorthand is a Database Management System in shorthand technique.

  • There are no difference between MySQL & MySQL Shorthand tool, than a faster query.

  • To do so, shotforms are used as input queries by defining the well-known SQL
    syntax in the backend part of the query executed.

    • Why MySQL Shorthand?


  • Some of the Students / Developers repeatedly will have to modify data in a database with well-known SQL syntax.
  • This repetition will have common syntax usage again & again which becomes boring to type the entire syntax at some point
  • We know that SQL Syntax is not a bigger one but a simple syntax.
  • This tool is designed to make it even a shorter & simpler syntax.
  • Hence, this tool is made to reduce their work in typing & to have the same kind of data modification with minimized query syntax.

    • Which Datatype is used? VARCHAR

    VARCHAR is set as default datatypes to columns created for tables.

    As data in databases are extracted by other software programming languages, datatypes can be parsed to any required format of the programmer's choice if the data in database is stored as a VARCHAR datatype.
